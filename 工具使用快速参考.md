# 🛠️ 模板OCR工具使用快速参考

## 📝 标注工具

### 坐标标定工具
```bash
python3 tools/coordinate_calibrator.py 911-20251016.jpg
```

**功能**：标注数据区域坐标

**键盘操作**：
- **鼠标左键拖动** - 框选区域
- **Space（空格）** - 确认当前区域并继续
- **Z** - 撤销上一个标注
- **S** - 保存坐标到 JSON
- **L** - 加载已有坐标
- **Q** - 退出程序
- **+/-** - 放大/缩小图像
- **H** - 显示帮助

**输出**：`config/template_coordinates.json`

---

## 🧪 测试工具

### 1. 部分识别测试（最常用）
```bash
python3 test_partial_recognition.py
```

**功能**：测试当前已标注区域的识别效果

**输出**：
- 终端：识别结果列表
- `partial_recognition_results.csv`：识别结果数据

**何时使用**：
- 每次标注后验证效果
- 修复坐标后检查改进
- 阶段性检查标注质量

---

### 2. 完整识别测试
```bash
python3 test_template_ocr.py
```

**功能**：测试所有71个数据项的识别

**何时使用**：完成全部标注后的最终测试

---

### 3. 详细识别调试
```bash
python3 debug_recognition_detailed.py
```

**功能**：显示每个字段的详细识别过程

**输出信息**：
- OCR原始文本
- 清理后的文本
- 识别数值
- 范围验证结果
- 失败原因分析

**何时使用**：识别失败时，查找具体原因

---

### 4. 不同预处理模式测试
```bash
python3 test_different_preprocess.py
```

**功能**：测试7种不同的图像预处理方法

**测试方法**：
1. 原图
2. 灰度
3. Otsu二值化
4. 自适应二值化
5. 反色Otsu
6. 反色自适应
7. 形态学增强

**何时使用**：某些字段识别效果不好时，找最佳方法

---

## 🎨 可视化工具

### 1. 坐标可视化
```bash
python3 visualize_coordinates.py
```

**功能**：在图像上绘制所有标注区域

**颜色含义**：
- 🟢 绿色框 = 识别成功
- 🔴 红色框 = 识别失败

**输出**：`coordinate_visualization.png`

**何时使用**：
- 检查标注区域是否准确
- 找出失败区域的位置
- 批量检查标注质量

---

### 2. ROI图像查看器
```bash
python3 debug_roi_viewer.py
```

**功能**：提取并保存每个区域的ROI图像

**输出**：`debug_rois/` 文件夹
- 每个文件包含：原图 | 灰度 | 二值化
- 可以看到OCR"看到"的内容

**何时使用**：
- 检查预处理效果
- 分析识别失败原因
- 调整预处理参数

---

### 3. 失败坐标查看器
```bash
python3 fix_failed_coordinates.py
```

**功能**：逐个放大显示识别失败的区域

**交互方式**：
- 按任意键：下一个
- ESC：退出

**何时使用**：修复失败区域前，先查看问题所在

---

## 📊 工作流程推荐

### 流程1：首次标注
```
1. coordinate_calibrator.py      # 标注坐标
   ↓
2. test_partial_recognition.py   # 测试识别
   ↓
3. visualize_coordinates.py      # 查看成功/失败区域
   ↓
4. fix_failed_coordinates.py     # 检查失败区域
   ↓
5. coordinate_calibrator.py      # 重新标注失败区域（按L加载）
   ↓
6. test_partial_recognition.py   # 再次测试
```

---

### 流程2：调试识别问题
```
1. test_partial_recognition.py          # 发现问题
   ↓
2. debug_recognition_detailed.py       # 查看详细原因
   ↓
3. debug_roi_viewer.py                 # 查看ROI图像
   ↓
4. test_different_preprocess.py        # 测试不同预处理
   ↓
5. 修改 template_ocr_extractor.py     # 调整预处理策略
   ↓
6. test_partial_recognition.py         # 验证改进
```

---

### 流程3：批量标注
```
1. coordinate_calibrator.py      # 标注一批（如8-16项）
   ↓
2. test_partial_recognition.py   # 立即测试
   ↓
3. 成功率 > 90%？
   YES → 继续下一批
   NO  → visualize + fix → 重新标注 → 再测试
```

## 💡 使用技巧

### 技巧1：批量测试

- 每次标注 8-16 项
- 立即运行 `test_partial_recognition.py`
- 确保成功率 > 90% 后再继续

cp config/template_coordinates.json config/template_coordinates_backup_$(date +%Y%m%d).json

## ❓ 常见问题

**Q: 哪个工具最重要？**  
A: `test_partial_recognition.py` - 每次标注后都要运行

**Q: 标注时如何确保精确？**  
A: 
1. 使用 `+/-` 放大图像
2. 框选时完全包含数字但不要太大
3. 标注后立即测试

**Q: 识别失败了怎么办？**  
A: 
1. 先运行 `visualize_coordinates.py` 看哪些失败
2. 再运行 `debug_recognition_detailed.py` 看失败原因
3. 使用 `coordinate_calibrator.py` 重新标注

**Q: 如何知道标注质量好不好？**  
A: 识别成功率 > 90% = 优秀，> 80% = 良好，< 70% = 需要改进
