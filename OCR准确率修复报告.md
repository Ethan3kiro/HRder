## 🔍 问题根因分析

### 发现的问题

通过对比历史测试结果 (`test_20251111_results.csv`)，发现当前版本缺失了9个历史可识别的项目：

**缺失项目**:
- ABCD = 28.0 °C
- Z-Plane-A-Current-1 = 7.2 A
- Z-Plane-A-Current-4 = 7.2 A
- Z-Plane-A-Current-5 = 7.3 A
- Z-Plane-A-Current-7 = 7.8 A
- Z-Plane-B-Current-8 = 7.5 A
- Z-Plane-C-ISOTemp-1 = 39.0 °C
- Z-Plane-C-ISOTemp-3 = 38.0 °C
- Z-Plane-D-ISOTemp-8 = 42.0 °C

### 根本原因

通过直接OCR测试发现，问题出在**预处理方法**上：

#### 使用不同预处理方法的识别结果对比：

| 项目 | 灰度图 | 简单二值化 | 自适应二值化 | 原代码使用 |
|------|--------|-----------|-------------|----------|
| ABCD | "23" | **"28"** ✅ | ".28" ❌ | 自适应 |
| Z-Plane-A-Current-1 | "" | **"72"** ✅ | "2." ❌ | 灰度 |
| Z-Plane-A-Current-7 | **"78"** ✅ | **"78"** ✅ | "7" ❌ | 灰度 |
| Z-Plane-C-ISOTemp-1 | **"39"** ✅ | **"39"** ✅ | "33" ❌ | 自适应 |

**结论**: 
- 温度字段使用**自适应二值化**导致识别率下降
- 电流字段使用**灰度图**对某些区域不够鲁棒
- **简单二值化 (Otsu)** 对两种字段都有最好的识别效果

#### 修复后:

```python
def _preprocess_for_current(self, roi: np.ndarray) -> np.ndarray:
    """Current字段专用预处理"""
    if len(roi.shape) == 3:
        gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
    else:
        gray = roi.copy()
    
    # ✅ 使用简单二值化 (Otsu自动阈值)
    _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
    return binary

def _preprocess_for_temperature(self, roi: np.ndarray) -> np.ndarray:
    """Temperature字段专用预处理"""
    if len(roi.shape) == 3:
        gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
    else:
        gray = roi.copy()
    
    # ✅ 使用简单二值化 (Otsu自动阈值) - 比自适应更稳定
    _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
    return binary
```

### 核心改进

1. **统一使用 Otsu 二值化**
   - 自动计算最佳阈值
   - 对不同光照条件更鲁棒
   - 比自适应二值化更稳定

2. **移除灰度图直接识别**
   - 灰度图在某些区域对比度不足
   - 二值化后字符边缘更清晰
